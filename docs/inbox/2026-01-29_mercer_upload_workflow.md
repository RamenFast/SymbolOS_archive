## The Upload Sequence: 🕯️✨🫴🌀✅🌠☂️

This document describes the Mercer upload workflow, encoded in the emoji sequence above.

### Workflow Breakdown

1. **🕯️ Candle (Illumination)**
   - Light the path: identify what needs to be documented
   - "Shine dat light" — trace the need back to primary cause
   - Question: What knowledge is being captured?

2. **✨ Sparkles (Clarity)**
   - Make it clear and legible
   - Structure emerges from understanding
   - Ti expression: organize the information

3. **🫴 Palm Up Hand (Offering)**
   - Present the documentation
   - Consent-forward: offering, not imposing
   - Ready for intake into the system

4. **🌀 Cyclone (Process)**
   - The intake ritual begins
   - Processing through the system
   - Movement toward integration

5. **✅ Check Mark (Validation)**
   - Verification complete
   - Quality gates passed
   - Ready for final step

6. **🌠 Shooting Star (Achievement)**
   - Milestone reached
   - Documentation successfully staged
   - Visible in the constellation

7. **☂️ Umbrella (Protection)**
   - Default-private scope applied
   - Safety gates in place
   - Protected under the umbrella

## Umbrella Coming Down (Over This Part!)

The phrase "umbrella coming down" means applying the **default-private constraint** to the uploaded documentation.

Key principles:
- ☂️ Umbrella = default-private scope
- Documentation starts protected by default
- Explicit decision required to make content public
- Safety-first approach to information sharing

From [poetry_translation_layer.md](../poetry_translation_layer.md):
> **☂️ Umbrella:** default-private scope; protect what matters

## Usage in Practice

When Mercer processes an upload request:

1. **Intake** → Place in `docs/inbox/` with date prefix
2. **Format** → Add provenance header
3. **Review** → Validate against safety constraints
4. **Classify** → Mark as public/internal/confidential/restricted
5. **Route** → Move to appropriate location when curated
6. **Protect** → Apply umbrella constraint (default-private)
